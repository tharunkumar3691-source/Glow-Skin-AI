Create a full-stack web application called GlowSkin AI.

GOAL
GlowSkin AI is an AI-powered skincare assistant that analyzes a user's face image, detects skin issues, predicts recovery time, and provides personalized skincare advice using environmental data and AI reasoning.

---

TECH STACK

Frontend: React with modern responsive UI
Backend: Python FastAPI
Image processing: OpenCV and Pillow
AI models: Amazon Bedrock Nova Lite and Nova Sonic
Database: SQLite
Charts: Chart.js for visualization

The backend must read AWS credentials from environment variables (DO NOT hardcode them).

Environment variables:

AWS_ACCESS_KEY_ID
AWS_SECRET_ACCESS_KEY
AWS_REGION

AWS_REGION must be set to:

us-east-1

---

FEATURE 1 — SKIN SCAN

Allow users to:

• Take a selfie using webcam
• Upload an image from device
• Optionally select skin type

Skin type options:

Dry
Oily
Combination
Balanced

The backend analyzes the face image and detects these skin conditions:

Acne
New acne
Post-acne marks
Dark spots
Dark circles
Dry skin
Puffy eyes
Rosacea
Wrinkles
Skin firmness

Use OpenCV to estimate these issues.

Return the detected skin issues to the frontend.

FEATURE 2 — SKIN HEALTH SCORE

Generate a Skin Health Score from 0–100.

Score meaning:

90–100 → excellent skin
70–89 → minor issues
50–69 → moderate issues
30–49 → visible problems
0–29 → severe issues

Display the score visually using a progress bar and gauge chart.

---

FEATURE 3 — AI TREATMENT GENERATION

Send detected skin issues to Amazon Bedrock Nova Lite.

Example prompt:

"User skin analysis detected acne, dark spots, and dry skin. Generate skincare routine, natural remedies, one home remedy, and recommended skincare products."

Return structured results:

Lifestyle advice
Morning skincare routine
Night skincare routine
Natural treatments
One home remedy

Recommended product types:

Face wash
Moisturizer
Sunscreen
Treatment cream

FEATURE 4 — RECOVERY PREDICTION

Predict estimated recovery time using the skin health score.

Rules:

80–100 → 11 days
60–79 → 16 days
40–59 → 21 days
20–39 → 30 days
0–19 → more than 30 days

Also display expected improvement percentages.

---

FEATURE 5 — ENVIRONMENTAL SKIN RISK PREDICTOR

Create a component called Environmental Skin Risk Predictor.

Steps:

1. Ask the user for their city or detect location automatically.
2. Fetch environmental data from a weather API such as OpenWeatherMap.

Collect data:

UV index
Humidity
Temperature
Air pollution (PM2.5)

3. Based on environmental conditions and detected skin issues, generate a Skin Risk Forecast.

Example risks:

High UV → dark spots may worsen
High pollution → acne inflammation risk
Low humidity → dry skin risk

4. Display a 7-day Skin Risk Forecast.

Example output:

Day 1: High UV risk → wear SPF 50 sunscreen
Day 2: High pollution → cleanse face twice
Day 3: Low humidity → apply moisturizer

5. Generate preventive skincare advice for the next 24 hours.
